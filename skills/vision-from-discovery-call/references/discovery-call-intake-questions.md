# Discovery Call Intake Questions

Source: `Discovery_Call_Intake_Form_BLANK.docx`

Ask every item below in order. Do not skip any item unless the user explicitly answers "skip".

1. Date of discovery call.
2. Call type. Choose one: Video; Phone; In-person.
3. Full name.
4. Business / company name.
5. Email address.
6. Phone / WhatsApp.
7. Time zone.
8. How did you find us?
9. Preferred communication method. Check all that apply: Email; Slack / Teams; WhatsApp; Upwork messages; Video calls only.
10. Project name / working title.
11. What does this app do in one sentence?
12. What problem does this solve?
13. Who are the end users? Include role, technical level, age group, and how often they will use the app if known.
14. App type. Check all that apply: SaaS platform; Client / customer portal; E-commerce; Internal business tool; Mobile web app; Marketplace; Data dashboard; Other.
15. Must-have features for launch (MVP). List every feature required on day one.
16. Nice-to-have features for Phase 2 or the future.
17. User roles and permission levels. Check all needed: Admin; Standard user; Guest / viewer; Manager / supervisor; Super admin; Client-facing role; Staff / internal role; Custom.
18. Third-party integrations needed. Check all that apply: Stripe / payments; Twilio / SMS; SendGrid / email; Google Maps; Google Auth / OAuth; QuickBooks; Zapier / automation; Salesforce; Slack; AWS S3 / storage; Calendar / scheduling; Other below.
19. Other integrations not listed above.
20. Design style preference. Choose one: Clean and minimal; Bold and colorful; Corporate and professional; Fun and playful; Dark mode / techy; No preference.
21. Existing brand assets. Check what you have: Nothing yet, needs full branding; Logo only; Logo plus color palette; Full brand guidelines / Figma.
22. Competitor or inspiration URLs.
23. Colors / fonts to avoid.
24. Anything you specifically do not want in the design.
25. Technology stack preference. Check all that apply: React; Next.js; Vue / Nuxt; Node.js; Python / Django; PostgreSQL; MySQL; MongoDB; No preference; Match existing stack.
26. Hosting preference.
27. Expected concurrent users at launch.
28. Mobile / device requirement. Choose one or more if needed: Responsive web, mobile-friendly; Mobile-first design; Desktop only is fine; Native iOS app also needed; Native Android app also needed; PWA, installable web app.
29. Does the app handle sensitive data? Check all that apply: No sensitive data; Personal info (PII); Financial / payment data; HIPAA / medical records.
30. Existing systems or codebase to integrate with. Include databases, APIs, or software this app needs to connect to or replace.
31. Desired launch date.
32. Is this a hard deadline? Choose one: Yes, with reason; Flexible.
33. Budget range. Choose one: Under $2,500 MVP / prototype; $2,500 to $7,500 small web app; $7,500 to $15,000 mid-size build; $15,000 to $30,000 full platform; $30,000+ enterprise / complex; Not sure yet, needs scoping first.
34. Payment preference. Choose one: Fixed price per milestone; Hourly rate; Monthly retainer; 50% up front / 50% on delivery.
35. Post-launch support needed? Choose one: 30-day bug fix period only; Ongoing monthly maintenance; On-call as needed; Full handoff, no support.
36. Red flags or scope concerns noted. Examples: scope creep risk, vague requirements, decision-maker not on call, unrealistic timeline.
37. Client's biggest fear or risk. Examples: budget overrun, missing launch date, quality.
38. How will the client define success?
39. Agreed next steps from this call.
40. Overall vibe of the call.
41. Likelihood to close. Choose one: High; Medium; Low.
42. Client signature or confirmation.
43. Developer / date.
